Ωpm omega package manager

an npm variant featuring signed packages and identical builds

0.0.5 - what is Ωpm today?

Signed

A PGP signature verifies that the code you download matches what the developer wrote. OmegaPM can't fake the signature.

Get signed update messages for your app's packages by running 'omegapm status'

Restricted

NPM runs arbitrary commands during the install process (that can be dangerous!). OmegaPM won't run scripts, even to compile code.

That doesn't stop you or the package from running fs.delete later on... but it does keep the install clean.

Identical

The code on your local machine, your server, the Ωpm website, GitHub, and the developer's machine ought to be the same. With JS-only packages, we can have consistent, reproducible builds.

Node's require is replaced with a custom function that checks that packages' SHAs match every time you run your server.

Installing

CLI

You install OmegaPM and Keybase using NPM. A Keybase account is optional. 
npm install omegapm 
npm install keybase-installer -g
keybase-installer
keybase config

Packages

See what's available, and add new packages.

Developer-signed messages

 
# install and verify a simple package
omegapm install omega-sqrt


0.0.6+ - what's next?

Open Standards

Ωpm should be able to download packages from a regular git repo, if its commits are signed using 'git commit -S'. It should be able to accept packages signed with gpg-zip or other open signing systems.

Beating Metadata

Padding and encrypting downloads would keep third parties from knowing what you're downloading.

The first step would be storing code repos on Ωpm servers, which comes with its own scaling and legal issues.

HTTPS and Tor

I launched before becoming HTTPS-ready. Got to work on that.

You could also use Tails today, but looking forward, Ωpm should have its own hidden service on Tor.
